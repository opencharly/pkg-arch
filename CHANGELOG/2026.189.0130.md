# 2026.189.0130 — pin makepkg git sources to #commit (detached-HEAD-proof)

The two `git+file://` sources (superproject + the sdk submodule) relied on the
clone's `origin/HEAD` symref: makepkg's working-copy step runs
`git checkout -B makepkg origin/HEAD`, which fails with
`fatal: invalid reference: origin/HEAD` whenever the cloned repo is on a
DETACHED HEAD — the sdk submodule detaches on every `git submodule update` /
gitlink-driven checkout, so the standalone/in-guest localpkg build broke
deterministically (surfaced as the `check-charly-vm` / `check-cachyos-vm`
"Creating working copy of opencharly-sdk … fatal: invalid reference:
origin/HEAD" failures under the concurrent bed roster). Pinned each source to
its working-tree HEAD via `#commit=$(git -C … rev-parse HEAD)`: makepkg now
checks out the exact commit, never `origin/HEAD`, and the build is reproducible
against the precise superproject + sdk commits. Verified: the pinned-commit
checkout succeeds against a detached sdk clone where the origin/HEAD checkout
fails, and `task build:charly` installs end-to-end.
