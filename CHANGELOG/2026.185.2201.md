# PKGBUILD: build Shape A host-command plugins from ./cmd/serve, not the root

The plugin-shape unification made every plugin candy's root package an importable
LIBRARY whose runnable entrypoint is the ./cmd/serve shim. The PKGBUILD built each
host-command plugin with `go build -o <bin> .`, which for a library package emits a
non-executable ar archive — so the installed /usr/lib/charly/plugins/<plugin> was a
.a archive and charly's out-of-process command dispatch failed with "exec format
error" (masked only for compiled-in plugins like preempt, whose broken installed
binary is never exec'd). The build now auto-detects ./cmd/serve when present (else
the root), mirroring the host buildPluginBinary target selection (plugin_loader.go).
Verified: `charly vm list` / `charly doctor` dispatch out-of-process again;
check-k3s-vm + the local command beds PASS.
