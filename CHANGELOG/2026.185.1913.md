# Wire the sdk file:// submodule as a second makepkg source

The plugin surface extracted to github.com/opencharly/sdk (mounted as the sdk/
submodule of the superproject). The standalone/AUR build resolves every candy's
`replace github.com/opencharly/sdk => ../../sdk` (and charly/go.mod's `=> ../sdk`)
against the makepkg clone, so sdk/ must be populated. makepkg's git+file:// clone
of the superproject does not fetch submodules, so a second `opencharly-sdk`
file:// source + a prepare() step wires it in as the sdk submodule
(git -c protocol.file.allow=always submodule update sdk).
